Mongo Document¶

The bread and butter on my bots. The Document class which can be found here

General setup (Outside of BotBase)¶

1from bot_base.db import Document
2
3# Create a Mongo connection
4conn = AsyncIOMotorClient(connection_url)
5db = conn["my_database"]
6
7# This will create a Document called "config"
8# on the "my_database", Mongo database.
9config: Document = Document(db, "config")

General setup (Using BotBase)¶

Want to know how to get autocomplete? You will need to follow the Converters in BotBase section, just with our without converters. Its the exact same process.

 1from bot_base import BotBase
 2
 3bot = BotBase(
 4    command_prefix="t.",
 5    mongo_url=os.environ["MONGO_URL"], # Your connection url
 6    mongo_database_name="my_bot", # The name of the database to use

 8
 9# ALL Documents are accessed via
10# self.db.document_name

12bot.db.config
13
14# Except it's automatically setup so you can start using it
15await bot.db.config.insert({"_id": ctx.guild.id, "prefix": "!"})

General Usage¶

Theres more to it, for that I’d recommend checking out the source code here

 1# ``document_id`` is either the `_id` field
 2# or a Dictionary which acts as the filter
 3
 4# To fetch a document using its `_id` field
 5# This can be None if that Document doesn't exist
 6data = await config.find(document_id)
 7
 8
 9# To insert data
10await config.insert({"_id": document_id, "prefix": "!", "guild_name": "testing"})
11data = await config.find(document_id)
12print(data) # -> {"_id": document_id, "prefix": "!", "guild_name": "testing"}
13
14# To update data, this overwrites EVERYTHING
15await config.update({"_id": document_id, "prefix": "?"})
16data = await config.find(document_id)
17print(data) # -> {"_id": document_id, "prefix": "?"}
18
19# The BETTER way to insert/update data is UPSERT
20# This will INSERT if required, otherwise it will UPDATE
21# This way you do not need to worry if it exists.
22await config.upsert({"_id": document_id, "prefix": "!", "guild_name": "testing"})
23data = await config.find(document_id)
24print(data) # -> {"_id": document_id, "prefix": "!", "guild_name": "testing"}
25
26
27# If you want to only update ONE field,
28# then you should use ``update_field_to`` to save
29# needing to fetch the data before
30await config.update_field_to(document_id, "prefix": ".")
31data = await config.find(document_id)
32print(data) # -> {"_id": document_id, "prefix": ".", "guild_name": "testing"}
33
34# This is equivalent and recommend over the following
35data = await config.find(document_id)
36data["prefix"] = "="
37await config.upsert(data)
38# Which achieves the same results, but double the db queries
39# If you need to update more fields, its up to you which you pick
40
41
42# Lets say we want to find ALL guilds with the prefix "?"
43# This will find and return a list of all guilds with
44# the field "prefix" as a "?", note this dict can be anything
45guilds = await config.find_many_by_custom({"prefix": "?"})

47
48# Delete all the data for a guild
49await config.delete(document_id)

51

53# Increment (Or decrement by passing a negative)

Converters¶

The Document class features the ability to use converters, so rather then get Dict’s back you get class instances.

Dynamically created Document’s will not use converters, if you wish to use them then your MongoManager should have them manually defined via a subclass and disabling the built-in manager class.

 1from typing import Any
 2
 3class Options:
 4    """
 5    Our database is:
 6
 7    {
 8        "_id": Any,
 9        "guild_id": int
10    }
11    """
12    def __init__(self, _id, guild_id):
13        self._id: Any = _id
14        self.guild_id: int = guild_id

Then when we wish to use it we can do:

1options_db: Document = Document(
2    self.db, "options", converter=Options
3)
4
5my_option: Options = await options_db.find(document_id_here)
6print(f"This option is for {my_option.guild_id}")

Converters in BotBase¶

Defining converters on built-in documents is a tad tricky.

 1# First create a subclass of MongoManager
 2from bot_base.db import Document, MongoManager
 3
 4class AmbroseMongoManager(MongoManager):
 5    def __init__(self, connection_url, database_name):
 6        super().__init__(connection_url=connection_url, database_name=database_name)
 7
 8        # Now define any documents you want explicit
 9        self.my_option: Document = Document(
10            self.db, "options", converter=Options
11        )
12
13# Now subclass BotBase
14from bot_base import BotBase
15
16class Ambrose(BotBase):
17    def __init__(self, *args, **kwargs):
18        # This gives us autocomplete!
19        self.db: AmbroseMongoManager = AmbroseMongoManager(os.environ["MONGO_URL"])
20
21        # leave_db means BotBase won't override our database
22        super().__init__(*args, **kwargs, leave_db=True)
23
24
25# Now we can use it!

27    command_prefix="t.",
28    mongo_url=os.environ["MONGO_URL"], # Your connection url
